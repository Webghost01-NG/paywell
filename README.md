# ✦ StellarRead — Pay-Per-Article on Stellar

> **Hackathon submission for [Stellar Agents × x402 × Stripe MPP](https://dorahacks.io/hackathon/stellar-agents-x402-stripe-mpp/detail)**

A pay-per-read article platform where **every unlock is a real Stellar testnet transaction**. No accounts, no subscriptions — just sign a micropayment with Freighter and the article unlocks instantly, AI content generated by Gemini.

---

## 🎯 What It Does

1. **Browse** 6 locked AI-written articles on crypto, Stellar, and the agent economy
2. **Connect** your Freighter wallet (Stellar testnet)
3. **Pay 1 XLM** → Freighter signs a real Stellar transaction
4. **Article unlocks** → Gemini AI generates the full content on the spot
5. **Verify** the tx hash directly on Stellar Expert

This demonstrates the **x402 payment flow** end-to-end:
```
CLIENT → requests article
SERVER → HTTP 402 + payment instructions (simulated in React)
CLIENT → pays via Stellar (Freighter signs tx)
STELLAR → settles in < 5 seconds
SERVER → delivers content (Gemini generates it)
```

## 🚀 Quick Start (Local Dev)

### Prerequisites
- Node.js 18+ 
- [Freighter wallet](https://freighter.app) browser extension

### 1. Clone & install
```bash
git clone https://github.com/YOUR_USERNAME/stellar-paywall
cd stellar-paywall
npm install
```

### 2. Configure environment
```bash
cp .env.example .env
```

Edit `.env`:
```env
# Get free API key at https://aistudio.google.com/apikey
VITE_GEMINI_API_KEY=your_gemini_api_key_here

# Your Stellar TESTNET wallet address (the one that RECEIVES payments)
# Create one at https://laboratory.stellar.org or use the default
VITE_STELLAR_RECEIVER=GCEZWKCA5VLDNRLN3RPRJMRZOX3Z6G5CHCGFPKPW9CBSHP2GPYUKMLA

# XLM amount per article unlock
VITE_UNLOCK_PRICE=1
```

> **Note:** If you skip the Gemini API key, the app still works — it uses a built-in mock article so you can test the full Stellar payment flow.

### 3. Run dev server
```bash
npm run dev
```

Open [http://localhost:5173](http://localhost:5173)

## 🔧 Using the App

### Step 1: Install Freighter
Download from [freighter.app](https://freighter.app). After install:
- Open Freighter → Settings → Network → Switch to **Testnet**

### Step 2: Fund your testnet wallet
Click **"🪣 Get XLM"** in the header after connecting. This calls [Friendbot](https://friendbot.stellar.org) and gives you 10,000 test XLM free.

### Step 3: Connect & unlock
- Click **"⚡ Connect Wallet"**
- Click any article's **"⚡ Pay 1 XLM to Unlock"** button
- Freighter pops up → **Approve** the transaction
- Article unlocks + Gemini generates content (~5 seconds)

### Step 4: Verify on-chain
Each unlock shows a green TX hash. Click it to view the transaction on [Stellar Expert Testnet](https://stellar.expert/explorer/testnet).

## 🔑 Stellar x402 Flow (Technical)

The payment happens in `src/utils/stellar.js → payWithFreighter()`:

```js
// 1. Load sender account from Horizon
const account = await server.loadAccount(senderPublicKey)

// 2. Build XLM payment transaction
const tx = new TransactionBuilder(account, { fee: BASE_FEE })
  .addOperation(Operation.payment({
    destination: RECEIVER_ADDRESS,
    asset: Asset.native(), // XLM
    amount: '1',
  }))
  .addMemo(Memo.text(`unlock:${articleId}`))  // x402 memo
  .setTimeout(60)
  .build()

// 3. Sign with Freighter (user approves in extension)
const signedXDR = await window.freighter.signTransaction(tx.toXDR(), { networkPassphrase })

// 4. Submit to Stellar testnet
const result = await server.submitTransaction(signedTx)
// result.hash = your on-chain receipt
```

## ⚠️ Known Limitations / Mocked Parts

| Feature | Status | Notes |
|---|---|---|
| Stellar payment | ✅ Real | Actual testnet transactions |
| Transaction verification | ✅ Real | Queries Stellar Horizon API |
| x402 HTTP server | 🟡 Simulated | Flow mimicked in React; no Express backend |
| Content generation | ✅ Real (if API key set) | Gemini 2.0 Flash |
| Content generation | 🟡 Mock fallback | Built-in article if no API key |
| Unlock persistence | 🟡 localStorage | Clears on browser data clear |

The x402 HTTP 402 challenge-response is simulated client-side (React handles the "server" logic). A production version would have a real Express/Node backend serving 402 responses.
